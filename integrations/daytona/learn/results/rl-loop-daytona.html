flywheel · reinforcement learning on agent trajectories

An agent improving from its own graded runs — RL, at the context layer.

◑ reference agent · REAL Daytona grading · no LLM spend

Same loop as reinforcement learning: the agent attempts tasks (rollouts), earns a verifiable reward from hidden tests, and its policy improves from its own trajectories. The one difference from textbook RL — the update is a durable lesson carried in context, not a weight change. Measured on a sealed held-out set, with a 95% CI on every point.

What the agent tried, round by round

  1. R1Treat missing or null fields as expected input — read them defensively and default them; never index a key that may be absent.+25.0pp · kept
  2. R2Handle the empty-input case explicitly before the main logic; don't index element 0 of a possibly-empty sequence.+9.4pp · under noise, rejected
  3. R3Parse and validate input types explicitly; never assume a string is already a number before doing arithmetic.+9.4pp · under noise, rejected

Why this isn't a hockey-stick. The gains are front-loaded — the first lesson fixes the commonest failure, each next one is rarer and worth less (diminishing returns). Two lessons were rejected: their held-out gain fell under the 12.3pp noise band, so they weren't credited — you can't keep an improvement you can't distinguish from noise. That's why the curve plateaus below 100%, not because the agent is perfect. To push the plateau higher you lower the noise floor — more held-out tasks and more rollouts, i.e. more Daytona. A free plumbing proof: the code-writer used stored reference solutions, but the grading was REAL — real Daytona sandboxes running the hidden tests. Only the model was stubbed; swap --agent codex for a live run and this same pipeline spends tokens instead.
